"# Mobile UX/UI Overhaul Plan

Per your instructions, the desktop structure and design will remain completely untouched. This plan focuses exclusively on introducing mobile-specific layouts, improved touch targets, and UX enhancements via the `@media (max-width: 760px)` CSS query in `styles.css`.

## Proposed Changes

### 1. Touch Targets & Usability (Apple HIG Standards)
On mobile devices, elements that are too small or too close together cause accidental taps. 
- **Forms & Buttons:** I will force a `min-height: 48px` on all planner form inputs, select dropdowns, form buttons, and the destination tab buttons.
- **Navigation:** I will increase the padding around the `.main-nav` links (`padding: 1rem 1.25rem`) so they are much easier to tap while scrolling horizontally.
- **Chips:** I will increase the vertical padding of the `.chip` tags to ensure they meet the minimum comfortable touch area.

### 2. Readability & Zoom Prevention
- **Font Sizing:** iOS automatically (and annoyingly) zooms the page in if a form input has a font size smaller than `16px`. I will ensure all form inputs (`.search-box input`, `.planner-form input`, `.planner-form select`) have `font-size: 16px;` on mobile.

### 3. Mobile-Specific Grid Layouts
Currently, many of our grids default to a single massive column (`1fr`) on mobile, forcing the user to endlessly scroll down. I will create tighter, "app-like" mobile grids:
- **Intent Grid ("Who's coming with you?"):** Override the 1-column layout to a 2x2 grid (`grid-template-columns: repeat(2, 1fr)`). This keeps the options visible in a single viewport.
- **Themes Grid:** Override the 1-column layout to a 2x3 grid (`repeat(2, 1fr)`) so users can view 6 themes in the space of 3.
- **Stats Grid:** Override to a 2x2 grid instead of 4 vertical rows.

### 4. Scrollbar Hiding
Horizontal scrolling is great for mobile UX, but visible scrollbars look clunky. 
- I will hide the scrollbars for the `.main-nav` and `.chips` container using `::-webkit-scrollbar { display: none; }`
